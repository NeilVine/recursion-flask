Recursive File Structure

This web application is presented as a solution to NetPay's Test 1: Recursive File Structure

Pages

  • Search Database page provides a user interface to query the contents of the database.
  • REST API page queries the database and provides a JSON response containing search results.

Technologies

  • Python, Flask, Jinja, FlaskForm
  • Sqlite - database
  • Bootstrap - navigation bar and submit button styling

Notes

A directory structure is essentailly a tree, and this solution builds a tree, whether from the text file or from the database records. The tree is built using recursion. And, the construction of the tree is an example of the Composite design pattern.

The database stores only the minimum amount of information necessary. This improves maintainablility and makes it simple to move the portions of the tree. The downside to this is that processing is required to get the full path to a node/leaf. Hence, this will be slower than a solution storing the full path in each record

Within the database, each node is stored as a record with the following fields:

  1. Name - The name of the node.
  2. Id - A unique Id to allow for duplicate node names.
  3. Parent Id - The Id of the node's parent.
